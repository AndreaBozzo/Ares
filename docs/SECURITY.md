# Security Policy

## Reporting a Vulnerability

If you discover a security issue, please report it responsibly.

Preferred: open a GitHub Security Advisory for this repository.

If you cannot use Security Advisories, open a new issue with the prefix
`[SECURITY]` and include the details needed to reproduce the issue. Do not share
exploit code or sensitive data publicly unless requested.

## Supported Versions

Security fixes are applied to the `master` branch. If a release is tagged, the
latest release is considered supported.
